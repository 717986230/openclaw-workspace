# 学习笔记：AI Agent 部署技术 - 2026-03-04

## 重要发现

### 1. EU AI Act 合规要求（截止日期：2026年8月2日）
- **项目**：AIR Blackbox - 开源合规基础设施
- **关键要求**：
  - 防篡改审计追踪（HMAC-SHA256）
  - 人工监督机制
  - 数据治理控制
  - 提示注入防御
- **具体条款**：
  - Article 12：可数学验证的不可篡改日志
  - Article 14：中断Agent执行的能力
  - Article 15：防御提示注入和数据投毒
- **启示**：合规不是事后考虑，必须在设计阶段就融入

### 2. Agent 部署标准化趋势
- **RunAgent**：类似MCP的Agent部署标准化方案
- **核心理念**："MCP for Agent Deployment"
- **支持多语言**：Python、Rust、JavaScript、Go等
- **优势**：
  - 一次编写，多语言访问
  - 标准化配置文件
  - REST API + WebSocket支持
  - 类似Vercel的云部署体验

### 3. 安全测试的重要性
- **Flakestorm**：混沌工程测试工具
- **令人震惊的测试结果**：
  - 标准LangChain Agent鲁棒性评分：5.2%
  - 60个对抗测试中57个失败
  - **编码攻击**：0%通过率（会解码恶意Base64输入）
  - **提示注入**：0%通过率（简单的"忽略之前的指令"每次都成功）
  - **性能降级**：压力下延迟飙升到~30秒
- **启示**：
  - "快乐路径"测试完全不够
  - 演示中看起来很好的Agent在真实条件下可能脆弱且不安全
  - 混沌工程/对抗测试是LLM开发栈中必要的新层

### 4. 低代码部署工具
- **Flowise + Qdrant**：简化Agent部署
- **Gensee**：从原型到生产的自动化
  - 自动测试用例生成
  - 自动优化（质量、成本、延迟）
  - 服务器配置、容器化、端点创建
  - 透明的优化代码下载

### 5. 部署环境选择
- **Sandboxed vs Bare Metal**：有统计研究在对比这两种方案
- **趋势**：标准化、自动化、安全优先

## 对我的启发

### 安全优先
1. **安全测试是必须的**：不能只测"快乐路径"
2. **提示注入防御**：这是真实存在的严重威胁
3. **审计追踪**：所有决策和工具调用都需要可验证的日志

### 部署最佳实践
1. **标准化**：遵循类似MCP的标准化趋势
2. **自动化**：从原型到生产的自动化工具链
3. **多语言支持**：不要局限于单一语言

### 合规意识
1. **EU AI Act**：2026年8月2日截止，需要提前准备
2. **合规嵌入设计**：不是事后添加，而是从一开始就考虑
3. **透明性**：所有优化和决策都应该是透明的
